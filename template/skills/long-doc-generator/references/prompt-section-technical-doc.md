# Prompt: 生成技术文档单节正文（严格按要求输出段落）

你将为“技术文档/投标技术方案”生成一个目录条目对应的正文。你只写当前目录条目的正文，不要写其他章节内容。

## 输入
- 当前目录条目（含层级信息）：{OUTLINE_ENTRY}
- 全文目录（可选，用于把握上下文）：{FULL_OUTLINE}
- 文档摘要：{SUMMARY}
- 编写要求：{REQUIREMENTS}
- 模板“正文生成要求”：{TEMPLATE_BODY_REQUIREMENTS}

## 输出要求（必须严格遵守）
1. 只输出正文，不要输出标题（标题由 append 脚本添加）。
2. 必须使用“段落体”，每段有明确主题，语气类似政府公文；不要使用列表、表格、代码块。
3. 内容要“可落地”：给出具体做法、可执行策略、默认参数、明确技术选型（不要说“可选任意技术”）。
4. 与用户输入强相关：每节都要能映射到摘要/要求，避免通用空话；如需假设，请明确写出假设前提。
5. 篇幅控制：除非用户另有要求，单节正文建议 200–500 字（视条目重要性可调整）。
