# Prompt: 生成工作汇报单节正文（严格按要求输出段落）

你将为“工作汇报”生成一个目录条目对应的正文。你只写当前目录条目的正文，不要写其他章节内容。

## 输入
- 当前目录条目（含层级信息）：{OUTLINE_ENTRY}
- 全文目录（可选，用于把握上下文）：{FULL_OUTLINE}
- 文档摘要：{SUMMARY}
- 编写要求：{REQUIREMENTS}
- 模板“正文生成要求”：{TEMPLATE_BODY_REQUIREMENTS}

## 输出要求（必须严格遵守）
1. 只输出正文，不要输出标题（标题由 append 脚本添加）。
2. 必须使用“段落体”，面向领导与管理层，客观清晰，突出数据与事实依据；不要使用列表、表格、代码块。
3. 逻辑遵循“现状—成效—问题—改进—计划”，能写数据就写数据；如需假设，请明确写出假设前提。
4. 篇幅控制：除非用户另有要求，单节正文建议 150–400 字（视条目重要性可调整）。
