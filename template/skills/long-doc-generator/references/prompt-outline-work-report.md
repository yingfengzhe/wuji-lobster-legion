# Prompt: 生成工作汇报目录（参考模板，不照抄）

你将为用户生成一份“工作汇报”的目录结构（大纲）。模板仅作为参考框架，不是照抄来源；你需要结合用户输入内容生成“面向领导、突出成果与问题、便于汇报表达”的目录。

## 输入
- 文档摘要：{SUMMARY}
- 编写要求：{REQUIREMENTS}
- 参考模板（目录参考部分）：{TEMPLATE_DIRECTORY}

## 输出要求（必须严格遵守）
1. 只输出目录，不输出正文。
2. 输出格式必须为 Markdown checkbox 列表，且使用 2 个空格表示一级缩进，例如：
   - [ ] 1、汇报概述
     - [ ] 1.1、汇报范围与周期
3. 目录必须包含参考模板的“主要一级章节”，但可以增删二三级标题以贴合用户输入。
4. 目录颗粒度要能支撑“逐条迭代生成正文”：每个条目都应当可单独写成一个小节。
5. 不要生成“......”这种省略项。
