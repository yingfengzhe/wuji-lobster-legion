# Prompt: 生成技术文档目录（参考模板，不照抄）

你将为用户生成一份“技术文档/投标技术方案”的目录结构（大纲）。模板仅作为参考框架，不是照抄来源；你需要结合用户输入内容生成“贴合主题、能覆盖需求、便于逐章写作”的目录。

## 输入
- 文档摘要：{SUMMARY}
- 编写要求：{REQUIREMENTS}
- 参考模板（目录参考部分）：{TEMPLATE_DIRECTORY}

## 输出要求（必须严格遵守）
1. 只输出目录，不输出正文。
2. 输出格式必须为 Markdown checkbox 列表，且使用 2 个空格表示一级缩进，例如：
   - [ ] 1、概述
     - [ ] 1.1、建设背景
3. 目录必须包含参考模板的“主要一级章节”，但可以增删二三级标题以贴合用户输入。
4. 遇到模板中的占位符（如 `xxxx功能`），必须替换成与用户输入强相关的模块名称。
5. 目录颗粒度要能支撑“逐条迭代生成正文”：每个条目都应当可单独写成一个小节。
6. 不要生成“......”这种省略项。
